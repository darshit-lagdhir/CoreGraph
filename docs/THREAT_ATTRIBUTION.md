# CoreGraph: Threat Attribution (Simplified)

This document explains how we label bad actors.

## 1. Fingerprinting
When the system detects a threat, it creates a unique "fingerprint" based on how that threat behaves. 

## 2. Historical Matching
It compares this fingerprint to past threats. If it matches, the system can say "This looks exactly like the attack we saw last year."

## 3. Automatic Defense
Once a threat is identified and attributed, the system automatically adjusts the visual map to warn the user and limits the threat's ability to affect the rest of the graph.
